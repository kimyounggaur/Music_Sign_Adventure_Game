## 0-D. 사용법 (리마스터 작업 5규칙)

1. **그리기만 바꾼다.** 판정·퀘스트·세이브·조작 로직은 불변. AI가 로직을 손대면 "render 계층만"이라고 답한다.
2. **성능 게이트.** 디오라마·앰비언트는 프레임 비용이 있다 — 각 스텝 자가 테스트에 FPS 기준(55+)이 있고, Step 3의 자동 폴백이 최후 방어선이다.
3. **한 스텝 = 한 배포.** 스텝마다 실기기 확인 후 Vercel 푸시, 문제 시 해당 스텝만 롤백.
4. **세이브 신성불가침.** SAVE_KEY·version:1·스키마 유지(설정 추가는 defaultDataSafe 기본값 방식만).
5. **저작권 원칙(0-A) 상속.** 프롬프트에 "닌텐도 자산·식별 요소 금지, 오리지널로 치환"이 내장되어 있다 — AI가 하트나 루피를 그리려 하면 즉시 교정한다.

# PART A — 월드의 질감 (기초부터)

## Step 1. 타일 리마스터 — 장난감 자연 만들기

**왜:** 현재 drawTile은 체커보드 2색+단색 길·물이라 '프로토타입 회색박스' 인상이 남아 있다. 꿈꾸는 섬풍의 절반은 타일 질감(잔디 점묘, 길의 밝은 가장자리, 포말 이는 물)에서 나온다.

```text
[리마스터 — Step 1/10: 타일]

drawTile과 CONFIG를 증분 수정한다. 타일 코드 의미(0 풀/1·2 길/3 물/4·5 실내)는 불변.

## 1. LA_PALETTE 도입
- CONFIG.LA = { grass:'#7ECB5A', grassAlt:'#74C251', grassDot:'#4FA043', grassShade:'#3E8E41',
  path:'#EAD08F', pathEdge:'#F7E7B5', pathDot:'#D9B96F', waterDeep:'#3D9BE9',
  waterShallow:'#6FC0F2', foam:'#FFFFFF', cliff:'#8A5A3B', cliffTop:'#A9744C',
  cream:'#FFF1D6', wood:'#9C6B4A' } 추가. palettes.village/forest/tower의 값을 이
  팔레트 참조로 교체(마을=크림·우드 톤 유지하되 채도 상향).

## 2. 잔디(0) — 점묘 질감
- 체커보드 제거. base 단색 위에 타일당 고정 위치 점 3~4개(grassDot, 지름 2~3px 라운드)와
  잔디잎 1개(2px 세로선+살짝 기울임)를 그린다.
- 점 위치는 (x*7+y*13)%N 같은 좌표 해시로 결정(매 프레임 동일 — Math.random 사용 금지,
  프레임마다 지글거리면 실패).

## 3. 길(1·2) — 밝은 가장자리
- path 채움 후, 상하좌우 이웃이 길이 아닌 변에만 pathEdge 3px 라운드 스트립을 그려
  '봉긋한 흙길' 느낌. 타일당 pathDot 점 2개(좌표 해시).
- 이웃 판정을 위해 drawTile 시그니처에 map 접근이 필요하면 tileAt을 내부에서 사용.

## 4. 물(3) — 살아있는 수면
- waterDeep 채움 + 상단 이웃이 물이 아니면 물가 포말: 흰 라운드 스트립이
  sin(time*2 + x)로 1px 출렁임. 수면 반짝: 좌표 해시 위치에 흰 점 1~2개가
  time 기반으로 서서히 명멸(alpha 0~.5). 기존 물결 곡선은 제거.

## 5. 실내(4·5) — 마루 판자
- 가로 판자 라인(어두운 1px, 8px 간격)+판자마다 미세 명도 차. 탑 내부가 '오르골 내부'답게.

## 6. 절벽/벽 오브젝트 대비
- collision 전용 벽 타일이 그려지는 곳(현재 wall색 사각)은 cliffTop 윗면+cliff 몸통
  2톤+하단 어두운 윤곽 2px로 입체감(위에서 본 단차 문법 — 이후 2막 elevation의 기초).

## 자가 테스트
T1. 점·잎 위치가 프레임 간 완전 고정(지글거림 0) — 좌표 해시 증명
T2. 길 가장자리가 이웃 방향에만 정확히 생김(교차로·끝단 확인)
T3. 물가 포말이 물-뭍 경계에만, 출렁임이 부드러움
T4. 세 맵(마을·탑·숲) 모두 팔레트 전환 정상, 스크린샷 3장 비교 서술
T5. FPS 55+ (타일 루프 안에서 객체 생성·그라디언트 남발 금지 — 필요 시 색 캐시)

## 완료 기준: T1~T5 통과 → "✅ Step 1 완료" + 변경 함수 전체 출력
```

## Step 2. 엔티티 토이화 — 솔·NPC·소품의 캐릭터성

**왜:** 플레이어가 파란 사각형+점, NPC가 색 사각형+2글자다. 꿈꾸는 섬풍의 나머지 절반은 '큰 머리 토이 캐릭터 + 발밑 블롭 그림자 + 글로시 하이라이트'에서 나온다. 스프라이트 없이 전부 절차 드로잉으로 만든다.

```text
[리마스터 — Step 2/10: 캐릭터]

drawPlayer·drawObject를 증분 수정한다. 히트박스(PLAYER_W/H, objectRect)는 절대 불변 —
그림만 히트박스보다 커 보이게 그린다(발 기준 정렬).

## 1. 공통 유틸
- drawBlobShadow(cx, footY, w): 타원 rgba(30,40,30,.22). 모든 캐릭터·소품이 먼저 호출.
- drawGloss(cx, cy, r): 좌상단 하이라이트 소원 rgba(255,255,255,.35).
```
