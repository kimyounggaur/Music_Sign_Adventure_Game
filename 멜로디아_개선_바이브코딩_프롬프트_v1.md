### 0.4 개선 트랙과 실행 순서

```text
트랙 A (안정성 핫픽스)      A1 → A2 → A3        반나절     기대 동작을 먼저 확정
트랙 B (자동 테스트 인프라)  B1 → B2 → B3 → B4    1~2일      ★ 이 문서의 핵심

트랙 C (UX·콘텐츠 개선)     C1 → C2 → C3 → C4    1~2일      각 스텝마다 회귀 테스트 동반

```

> **왜 A가 먼저인가:** 테스트는 "올바른 동작"을 고정하는 작업이다. BUG-01~06을 고치기 전에 테스트를 쓰면 버그가 스펙으로 굳는다.
