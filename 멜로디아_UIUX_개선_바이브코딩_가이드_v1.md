## 2. 사용법 (개선 작업 5규칙 — 신규 개발과 다른 점 주의)

1. **전체 리팩토링 금지.** 모든 프롬프트는 "기존 index.html을 붙여넣고 해당 부분만 증분 수정"을 요구한다. AI가 구조를 갈아엎으려 하면 "기존 구조 유지, 지시한 diff만"이라고 답한다.
2. **세이브 호환 유지.** SAVE_KEY 'melodia_v01_save'와 version:1은 바꾸지 않는다. 새 설정 필드는 defaultDataSafe()에 기본값을 추가하는 방식으로만(기존 로드 머지 로직이 흡수).
3. **한 스텝 = 한 배포 단위.** 각 스텝 후 실기기(폰) 확인 → Vercel 푸시. 문제가 생기면 그 스텝만 되돌린다.
4. **자가 테스트 표 보고 후 진행.** "✅ Step N 완료" 출력 전 다음 스텝 금지.
5. **불변식 존중.** 1-A 표의 항목(상태 머신, 데이터 주도, 주파수 공식, 저장 어댑터, 조작 스킴)은 어떤 개선에서도 훼손 금지.
