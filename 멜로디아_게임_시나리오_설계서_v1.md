## 0. 설계 원칙 — 두 소스 문서의 결합 방식

1. **뼈대는 [개발문서]에서 가져온다.** 탑다운 2D 어드벤처 프레임, "작지만 끝까지 완성"(0단계), 데이터 주도 설계(대화·퀘스트·아이템 JSON), 퍼즐 공식(힌트→관찰→조작→피드백→보상), 전역 상태 관리, flag 기반 저장.
2. **살은 [기믹문서]에서 가져온다.** 임시표(♯/♭/♮) 지형 기믹, 쉼표 시간 제어, 슬래시 커맨드 팔레트 + 악보 타임라인 UI, 불협화음의 카타콤, 엇박자 골렘, 다카포 마왕 3페이즈.
3. **시점 통일: 탑다운.** [기믹문서]의 플랫포머식 기믹(점프·발판 높이)은 모두 탑다운의 **단차(절벽·계단)** 문법으로 변환한다. 점프 대신 "반 칸 단차는 오를 수 있음 / 한 칸 이상은 불가" 규칙을 쓴다.
4. **이론 정확성은 타협하지 않는다.** 모든 퍼즐의 정답은 실제 음악 이론과 일치한다(예: C장조 으뜸화음은 반드시 C·E·G, 조표는 전역·제자리표는 개별 적용). 화음·음계는 하드코딩하지 않고 근음 기준 인터벌 공식으로 생성한다.
5. **전체 시나리오를 설계하되, 개발은 프롤로그+1막(10~15분 분량)을 먼저 끝까지 완성**한다([개발문서] 0·20·21단계 원칙). 이후 막 단위로 확장한다.

## 1. 게임 개요

### 1.1 한 문장 콘셉트 ([개발문서] 1단계)

> 견습 음악사 '솔'이 마왕 다카포가 찢어버린 세계악보의 음악 기호들을 되찾아, 기호의 힘으로 퍼즐을 풀며 도둑맞은 '피네(Fine, 곡의 끝)'를 되찾는 2D 탑다운 어드벤처.

### 1.2 기본 정보

```text
게임 제목: 멜로디아 — 끝나지 않는 노래 (가제)
장르: 2D 탑다운 어드벤처 + 음악 기호 퍼즐
플랫폼: 모바일 웹 (세로 모드 기준, PC 브라우저 겸용)
플레이 시간: v1.0 기준 60~90분 / 첫 출시 버전(v0.1)은 10~15분
대상 유저: 음악 기초를 배우는 어린이~성인 초보자
교육 범위: 자리표, 음이름·오선, 임시표, 쉼표·음가, 셈여림·아티큘레이션,
          장/단 3화음, 반복·형식 기호(도돌이표·1/2번 괄호·코다·피네·D.C.)
핵심 재미 3가지: 탐험 + 기호 퍼즐 + 수집·복원(곡이 완성되어 가는 성취)
```

### 1.3 핵심 게임 루프 ([개발문서] 5단계)

```text
지역을 탐험한다
→ NPC에게 문제(멈춘 것·뒤틀린 것)를 듣는다
→ 흩어진 음악 기호를 찾아낸다 (악전 도감 카드 해금)
→ 기호를 오브젝트에 '부여'해 퍼즐을 푼다
→ 길이 열리고 세계악보의 한 파트가 복원된다
→ 마을 오르골의 곡이 한 겹 풍성해진다 (귀로 확인하는 보상)
→ 다음 지역이 열린다
```

### 1.4 엔딩 조건

5개 파트(멜로디·임시표·리듬·셈여림·화음)를 모두 복원하고, 다카포에게서 되찾은 '피네'를 세계악보 마지막 마디에 꽂으면 곡이 "처음부터 끝까지(D.C. al Fine)" 연주되며 엔딩.

## 2. 세계관과 스토리

### 2.1 배경 설정

- 세계 '멜로디아'는 중앙 오르골 탑에서 연주되는 거대한 악보, **세계악보(The World Score)**의 소리로 유지된다.
- 이 세계에서 음악 기호는 표기가 아니라 물리 법칙이다. ♯이 붙은 것은 실제로 반 칸 솟아오르고, 쉼표가 놓인 시간은 실제로 멈춘다.
- 지휘자 '마에스트로 카포'는 완벽한 연주에 집착하다 "곡이 끝나지 않으면, 실패한 연주도 없다"는 결론에 도달해 타락한다.

### 2.2 사건 (프롤로그 직전)

카포는 스스로를 '다카포(Da Capo, 처음으로)'라 칭하고, 세계악보에서 **피네(Fine)를 뜯어 봉인**한 뒤 곡 전체에 거대한 도돌이표를 걸어버린다. 그 충격으로 악보가 찢어지며 음악 기호들이 대륙 곳곳에 흩어지고, 기호를 잃은 세계는 지역마다 다른 병증을 앓는다 — 음이 사라진 숲, 반음이 뒤틀린 협곡, 박자가 폭주한 유적, 굉음이 멈추지 않는 산.

### 2.3 여정 시놉시스 (3부 구성)

| 부 | 내용 | 해당 챕터 |
|---|---|---|
| 1부 기호 회수 | 솔이 높은음자리표를 시작으로 4개 지역의 병증을 고치며 기호 파트를 복원 | 프롤로그~4막 |
| 2부 진실 | 불협화음의 카타콤에서 엇박자 골렘을 잠재우고, 다카포가 원래 세계 제일의 마에스트로였음을 알게 됨 | 5막 |
| 3부 곡의 끝 | 무한 콘서트홀에서 다카포의 반복 기호 권능을 역이용, 코다를 따라 피네에 도달. 다카포를 파괴하는 게 아니라 **곡을 끝내 줌으로써 구원**한다 | 최종막 |

**주제:** "노래는 끝이 있어야 완성된다. 끝은 실패가 아니라 완성이다."

### 2.4 왜 이 스토리인가 (교육 설계 관점)

- 초보자가 가장 헷갈리는 **반복 기호의 진행 순서**(도돌이표 → 1·2번 괄호 → 코다 → 피네)를 최종 보스의 정체성과 클라이맥스 동선으로 만들어 몸으로 체험시킨다.
- "기호가 모일수록 마을 오르골 곡이 완성"되는 구조는 기호 → 소리 → 음악의 관계를 귀로 증명한다.

## 4. 핵심 시스템 설계

### 4.1 기호 시스템 (게임의 심장)

- **획득:** 기호는 필드 오브젝트·상자·퀘스트 보상으로 획득. 획득 즉시 '악전 도감' 카드 팝업(실제 이론 1줄 + 게임 기능 1줄).
- **부여(Attach):** 팔레트에서 기호 선택 → 부여 가능한 오브젝트가 자동 하이라이트 → 탭(또는 드래그&드롭)으로 부여. 부여 즉시 해당 이론의 소리가 재생된다(♯=반음 위로 미끄러지는 소리).
- **회수(Detach):** 부여된 기호는 다시 탭해 회수·재사용. **기호는 소모품이 아니라 도구다.** 퍼즐 난이도는 기호 개수 제한으로 만든다(예: ♯ 1개로 계단 3단을 번갈아 오르기).
- **팔레트 UI:** 화면 하단 버튼 → 노션 슬래시 커맨드 스타일의 콤팩트 리스트 팝업([기믹문서] 채택). 보유 기호만 표시, 최근 사용 순 정렬.
- **악보 타임라인 UI:** 리듬 기믹 구간·보스전에서만 하단에 등장하는 1단 악보 바(4~8박). 쉼표·페르마타를 박 칸에 드래그해 세계의 시간을 편집한다([기믹문서] 채택). 평상시엔 숨겨 인지 부담을 최소화.

### 4.2 기호 ↔ 게임 기능 ↔ 이론 매핑 총괄표

| 기호 | 이론 의미 | 게임 기능 | 획득 시점 |
|---|---|---|---|
| 𝄞 높은음자리표 | 둘째 줄이 '솔', 음높이의 기준 | '악보의 눈': 숨은 오선·음표 가시화 + 기호 부여 능력 해금 | 프롤로그 |
| 음표 (도·미·솔 등) | 오선 위 위치 = 음높이 | 열쇠·발판: 오선 슬롯에 배치, 밟으면 실제 그 음 재생 | 1막 |
| ♯ 샵 | 반음 올림 | 대상을 반 칸 상승(단차 생성) | 2막 |
| ♭ 플랫 | 반음 내림 | 대상을 반 칸 하강 | 2막 |
| ♮ 제자리표 | 임시표·조표 효과 해제 | 대상만 기본 상태로 초기화(디스펠) | 2막 |
| 조표 | 곡 전체에 적용 | 맵 전역 룰(해당 음이름의 모든 오브젝트 변형) — 주로 '읽고 파훼할' 적의 권능 | 2막(개념) |
| 𝄽 4분쉼표 | 1박의 침묵 | 타임라인 1박 정지(함정 스킵) | 3막 |

| 스타카토 | 짧게 끊어서 | 짧게 튀는 연속 도약 이동·속공 타격 | 4막 |
| 슬러(레가토) | 부드럽게 이어서 | 끊긴 지형을 잇는 이음 다리 | 4막 |

| 𝄆 𝄇 도돌이표 | 구간 반복 | 구간 강제 루프(주로 적의 함정) | 최종막(개념) |
| 1·2번 괄호 | 1회차/2회차에 다른 진행 | 반복 회차에 따라 다른 경로가 열림 | 최종막 |
| ⊕ 코다 | 반복을 건너뛰어 후반부로 | 루프 탈출 워프 | 최종막 |
| Fine | 곡의 끝 | 최종 열쇠: 곡(보스전)을 종료시킴 | 최종막 |

**이론 검증 규칙(필수):** 화음 판정은 근음 기준 3도 쌓기 공식으로 생성·검증한다(장3화음 = 근음+장3도+완전5도). 방에 조표가 걸리면 해당 음이름의 모든 오브젝트 표기가 일치해야 하며, ♮ 부여 시 그 오브젝트만 예외 처리된다.

### 4.3 진행 보상 — 세계악보 복원

오르골 탑 내부에 '세계악보 복원도'를 상시 전시한다. 파트 복원 때마다 다음이 발생한다.

1. 복원 컷신: 되찾은 기호들이 악보로 날아와 박힌다.
2. 마을 BGM에 해당 파트가 추가된다. `멜로디만 → +임시표 장식음 → +리듬 반주 → +강약·표정 → +화음`
3. 탑에서 언제든 현재까지의 곡을 재생할 수 있다.

테마곡은 오리지널 8마디 곡 1개로 하고, 파트가 늘 때마다 같은 곡이 풍성해지는 것이 명확히 들리도록 5레이어 계층 편곡한다.

### 4.4 사운드 = 규칙 ([개발문서] 19단계 확장)

```text
정답 상호작용 → 협화음 / 오답 → 짧은 불협화음 (벌주지 않는 톤)
기호 부여 → 해당 이론의 사운드 (♯: 반음 상행 글리산도, ♭: 하행, ♮: 원음 복귀음)
음표 발판 → 밟는 순간 실제 그 음 재생 (오선 위치와 항상 일치)
화음 완성 → 실제 코드가 아르페지오 + 블록으로 재생
```

### 5.1 월드 맵 구조

```text

   │ 산 지하                    │
[불협화음의 카타콤](5막)   [반음 협곡](2막)
```

### 6.0 프롤로그 — 멈춘 오르골 (하모니아 마을)

**[개요]** 마을 맵 1개 / 3~5분 / 입장 조건 없음 / 획득: 높은음자리표 / 학습: 이동·상호작용·기호 부여 UX, "자리표는 음높이의 기준"

**[스토리]** 오프닝 컷신([개발문서] 25단계 구조): 평화롭게 연주되는 오르골 탑 → 하늘이 찢기고 악보 조각들이 유성처럼 흩어짐 → 오르골 침묵 → 솔이 집에서 깨어난다.

**[진행 단계]**

- **Step 1. 이동 튜토리얼** — 템포가 합류. "조이스틱으로 광장까지 가보자. 하나, 둘!" (이동만 허용, 다른 UI 잠금)

- **Step 3. 잠긴 탑 문 관찰** — 탑 문에 흐릿한 소용돌이 자국. 템포: "이 자국… 어디서 봤는데?" (관찰 단서)
- **Step 4. 첫 기호 획득** — 피치의 힌트 "우물 뒤에서 반짝이는 거 봤어!" → 우물 뒤 수풀에서 **높은음자리표** 획득. 획득 연출 + 도감 카드 팝업: "높은음자리표(𝄞): 소용돌이가 감는 둘째 줄이 '솔'이에요. 이제 숨은 오선이 보입니다."
- **Step 5. 첫 기호 부여 (조작 튜토리얼)** — 팔레트를 열어 높은음자리표를 탑 문에 부여(하이라이트 → 탭). 문 위로 오선 5줄이 빛나며 개방. *부여의 기본 문법을 여기서 완성한다.*
- **Step 6. 탑 내부 — 사건의 전모** — 찢어진 세계악보 대면. 남은 것은 거대한 도돌이표뿐. 다카포의 그림자 컷신: "곡이 끝나지 않으면, 실패한 연주도 없다." 클레프: "흩어진 기호들을 모아 파트를 복원해야 한다. 첫 기호들은 도레미 숲에 떨어졌어."
- **Step 7. 숲 문 개방** — flag `forest_gate_open` 설정. 자동 저장 안내 토스트.

**[보상 & flag]** 높은음자리표(영구 능력: 숨은 오선·음표 가시화), `tower_door_open`, `forest_gate_open`, MQ01 완료 → MQ02 시작.

### 6.1 1막 — 도레미 숲: 사라진 음표들 (v0.1의 본편)

**[개요]** 숲 맵 1개 / 8~10분 / 입장: `forest_gate_open` / 획득: 음표 도·미·솔, 오선의 인장 / 학습: 오선 위 위치=음높이, 낮은음→높은음, 기초 청음

**[스토리]** 음표를 잃은 숲은 새소리조차 없는 무음 지대. 라라가 입구에서 규칙을 노래한다: "낮은 소리에서 높은 소리로— 길은 노래를 따라 열리지." 세 음표를 되찾아 숲 안쪽 '오선 문'을 열면 멜로디 파트가 복원된다.

**[진행 단계]**

- **Step 1. 도(C) 회수 — 관찰형** — 피치의 목격담(마을에서 미리 들을 수 있음): "우물 두레박이 이상하게 무거웠어." 숲 초입 낡은 우물의 두레박을 상호작용으로 끌어올리면 **도 음표** 획득. 획득음으로 실제 '도'가 울린다.
- **Step 2. 미(E) 회수 — 청음 퍼즐 P101 '새의 노래'**
  - 힌트: 라라 "저 까마귀 코르부스는 제가 들은 노래를 따라 부르는 이에게만 마음을 열어."
  - 관찰: 나무 위 까마귀가 미 음표를 물고 있다. 아래에 종 발판 3개(각각 도·미·솔 소리).
  - 조작: 까마귀가 3음 멜로디를 재생(예: 솔–미–도) → 같은 순서로 종 발판을 밟는다.
  - 피드백: 한 발판씩 실음 재생, 순서가 틀리면 까마귀가 고개를 갸웃하며 다시 들려줌(무한 재시도, 페널티 없음).
  - 보상: 까마귀가 **미 음표**를 떨어뜨리고 어깨 위 단골 손님이 된다(장식 연출).
- **Step 3. 솔(G) 회수 — 지형 퍼즐** — 늙은 그루터기 속 상자를 바위가 막고 있다. 바위 2회 밀기(밀기 조작 학습)로 경로 개방 → **솔 음표** 획득.
- **Step 4. 관문 퍼즐 P102 '오선 다리'** ([개발문서] 14단계 예시의 확장)
  - 힌트: 표지판 "낮은 음에서 높은 음으로 길이 열린다."
  - 관찰: 강 위에 오선 5줄 모양 다리. 빈 슬롯 3개 — 아래 첫째 덧줄 / 첫째 줄 / 둘째 줄 위치.
  - 조작: 도(덧줄)·미(첫째 줄)·솔(둘째 줄)을 배치한 뒤 낮은음부터 순서대로 밟는다.
  - 피드백: 배치가 틀리면 그 슬롯이 '삐끗' 불협음과 함께 진동(즉시 재배치 가능). 순서대로 밟을 때마다 실음이 계이름 자막과 함께 울린다.
  - 보상: 세 음이 모두 울리면 C장조 으뜸화음(도·미·솔) 팡파르 → 오선 문 개방.
- **Step 5. 클리어 컷신 — 멜로디 복원** — 세 음표가 세계악보로 날아가 박히고, 마을 오르골이 8마디 멜로디를 처음으로 연주한다(마을 BGM 갱신). 클레프: "들리니? 노래가 돌아오고 있어." **오선의 인장** 수여.
- **숨김 요소:** 황금 음표 #1 — 폭포 뒤 샛길.

**[v0.1 임시 엔딩]** 첫 출시 버전은 여기서 "협곡의 문이 열렸다. 여정은 계속된다…" 메시지 + 복원된 멜로디 풀 재생으로 마무리([개발문서] 20단계: 짧은 엔딩 1개).

**[구현 힌트]** `noteTile.onStep → playPitch(note); if sequence == sorted(byPitch) → gate.open()` / 청음 퍼즐: `birdSong = shuffle([C,E,G]); compare(playerInput, birdSong)`.

### 6.2 2막 — 반음 협곡: 뒤틀린 조표 (임시표 편)

**[개요]** 협곡 맵 1개(3구역) / 10분 / 입장: `seal_staff` (세뇨 관문 ①) / 획득: ♯·♭·♮, 협곡의 인장 / 학습: ♯=반음 올림·♭=반음 내림·♮=해제, 조표는 전역·임시표는 개별

**[스토리]** 자수정 협곡의 수정 발판들이 뒤틀린 조표 폭풍 탓에 제멋대로 솟거나 꺼져 길이 끊겼다. 라라: "샵은 반 칸 위로, 플랫은 반 칸 아래로. 제자리표는… 원래 모습을 기억하게 하지."

**[진행 단계]**

- **Step 1. ♯ 획득** — 입구 제단에서 **샵** 획득. 도감 카드: "♯: 그 음을 반음 올려요. 이 세계에선 반 칸 솟아오르죠."
- **Step 2. P201 '반음 계단'** ([기믹문서] 기믹의 탑다운 변환)
  - 힌트: 템포 "반 칸이면 오를 수 있어. 한 칸은 무리야!"
  - 관찰: 절벽이 3단. 각 단 앞에 '파(F)' 수정 발판이 반 칸 낮게 박혀 있다.
  - 조작: F 발판에 ♯ 부여 → 반음 상행음과 함께 발판이 반 칸 상승(F♯) → 올라선 뒤 ♯을 회수해 다음 단의 F 발판에 재부여. ♯ 1개로 3단을 오르는 재사용 퍼즐.
  - 피드백: 부여/회수 때마다 반음 상행·복귀음. 발판 라벨이 F ↔ F♯로 실시간 변경.
  - 보상: 계단 위 상자에서 **♭ 획득**.
- **Step 3. P202 '내림 징검다리'** — 길을 막은 너무 높은 수정 기둥 2개에 ♭를 번갈아 부여해 반 칸씩 낮춰 징검다리로 만든다(♭ 재사용). 하행음 피드백.
- **Step 4. P203 '조표의 방'** ([기믹문서] 제자리표 디스펠)
  - 힌트: 방 입구 표지판에 조표 **♯ 1개**가 큼직하게 그려져 있다. 라라: "조표는 방 전체에 걸려. 하지만 제자리표는 딱 하나만 되돌리지."
  - 관찰: 방 안의 모든 'F 수정'이 가시 형태(F♯ 상태)로 융기해 통로를 막고 있다. 다른 음이름 수정은 멀쩡하다.
  - 조작: 피치의 목격담("가시가 없는 샛길이 있었어!")을 따라 옆 동굴에서 **♮ 획득** → 경로 위의 F 가시에만 ♮을 부여해 원래의 안전한 발판으로 초기화하며 전진(♮ 1개 재사용).
  - 피드백: ♮ 부여 시 '원음 복귀' 청명음 + 가시가 수정 발판으로 되돌아가는 애니메이션.
  - 보상: 출구 문의 확인 퀴즈 "이 방의 조표는 무엇이었을까?" (♯1 / ♭1 / 없음 3지선다) 정답 시 개방 — 관찰을 지식으로 굳히는 마무리.
- **Step 5. 클리어 컷신** — 임시표 파트 복원: 오르골 멜로디에 반음 장식음이 더해진다. **협곡의 인장** 수여.
- **숨김 요소:** 황금 음표 #2 — ♭로 낮춰야 보이는 웅덩이 바닥.

**[보상 & flag]** ♯·♭·♮(영구), `seal_accidental`, 임시표 파트 복원.

**[구현 힌트]** `attach('#', obj) → obj.height += 0.5tile; obj.pitch += 1semitone` / 조표의 방: `room.keySignature = {F:'#'} → spawn 시 모든 F 오브젝트를 sharp 상태로; attach('♮', obj) → obj.localOverride = natural`.

### 6.3 3막 — 멈춘 태엽 유적: 박자의 폭주 (쉼표 편)

**[개요]** 유적 맵 1개(4구역) / 10~12분 / 입장: `seal_accidental` (세뇨 관문 ②) / 획득: 4분·8분·온쉼표, 태엽의 인장 / 학습: 쉼표=침묵의 길이, 1박·반박·온마디, 박 세기

**[스토리]** 세계의 박자를 지키던 메트로놈 유적이 기호를 잃고 폭주 — 프레스와 레이저가 4/4 정박으로 쉼 없이 작동한다. 템포가 떨리는 목소리로: "여긴… 내 고향이야." (개인 서사 시작)

**[진행 단계]**

- **Step 1. 4분쉼표 획득 & 타임라인 UI 튜토리얼** — 부서진 메트로놈 잔해에서 **4분쉼표** 획득. 이때 처음으로 화면 하단에 **악보 타임라인 바(4박)**가 등장하고, 템포가 사용법을 안내한다: "여기 한 칸이 한 박이야. 쉼표를 꽂으면 그 박 동안 세계가 숨을 쉬어."
- **Step 2. P301 '프레스 복도'** ([기믹문서] 리듬 함정)
  - 힌트: 템포가 박자를 세어준다 "쿵(1)–약(2)–쿵(3)–약(4)!"
  - 관찰: 프레스 A는 1박, 프레스 B는 3박에 내리찍는다. 타임라인에 프레스 아이콘이 1·3칸에 예고 표시.
  - 조작: 4분쉼표를 타임라인 3박 칸에 드래그 → 다음 사이클에서 B가 1박 동안 정지 → 통과 → 쉼표 회수 → 1박 칸에 재부여 → A 통과.
  - 피드백: 쉼표가 발동하는 박에 세계의 소리가 실제로 '뚝' 끊긴다(오디오 덕킹). 실패해도 밀려날 뿐, 즉시 재도전.
  - 보상: 복도 끝 상자에서 **8분쉼표 2개** 획득.
- **Step 3. P302 '레이저 홀'** — 레이저가 2박과 2.5박(반 박 간격)에 연속 발사. 4분쉼표(1박)로는 한 발이 남는다 → **8분쉼표 2개**를 2박·2.5박 칸에 나눠 꽂아 반 박씩 두 번 침묵시킨다. "1박 = 반박 + 반박"을 손으로 체험.
- **Step 4. 온쉼표 획득 & P303 '디스코드 둥지 잠입'** ([기믹문서] 스텔스 존)
  - 힌트: 라라 "저 녀석들은 눈이 없어. 귀로만 쫓지."
  - 관찰: 보물방에서 **온쉼표** 획득. 디스코드 몬스터들의 소리 감지 반경이 링으로 시각화된다. 플레이어 발소리 게이지 존재.
  - 조작: 온쉼표를 바닥에 설치 → 반경 내 '침묵의 오라'(4박 지속) 생성 → 발소리가 0이 됨 → 오라에서 오라로 징검 이동하며 잠입. 지속이 끝나면 회수·재설치.
  - 피드백: 오라 안에서는 BGM까지 여려진다. 들키면 방 입구로 복귀(체력 개념 없음).
- **Step 5. P304 '박자 자물쇠 문'** — 문이 리듬 패턴(예: ♩ ♪♪ ♩ 𝄽)을 두드려 들려준다 → 같은 리듬으로 문을 탭. 판정 ±150ms. 성공 시 문이 같은 리듬으로 화답하며 개방(리듬 청음 마무리).
- **Step 6. 클리어 컷신** — 리듬 파트 복원: 오르골 곡에 반주 리듬이 생긴다. 유적의 메트로놈들이 일제히 규칙적으로 째깍인다. 이벤트: 유적 심장부의 **수호상 대좌가 비어 있다.** 템포: "수호상이… 없어. 누가 뜯어간 거야." (5막 복선) **태엽의 인장** 수여.
- **숨김 요소:** 황금 음표 #3 — 온쉼표 오라 안에서만 우는 벽 뒤 공간.

**[보상 & flag]** 쉼표 3종, `seal_rhythm`, 리듬 파트 복원, flag `guardian_missing`.

**[구현 힌트]** `timeline[i] = quarterRest → beat i 동안 trap.update() skip` / `wholeRestAura(pos, r): player.noiseLevel = 0 for 4 beats` / 자물쇠: `compareRhythm(input, pattern, tolerance=150ms)`.

### 6.4 4막 — 오르간 파이프 산: 굉음의 폭풍 (셈여림 편)

**[개요]** 산 맵 1개(4구역) / 10~12분 / 입장: `seal_rhythm` (세뇨 관문 ③) / 획득: p·f 배지, cresc./decresc., 스타카토, 슬러, 오르간의 인장 / 학습: 셈여림=소리의 크기, 점점 크게/작게, 짧게 끊기·부드럽게 잇기

**[스토리]** 산 전체가 거대한 파이프오르간. 셈여림 기호를 잃은 산은 모든 파이프가 f(포르테)로 고정돼 폭풍 같은 굉음과 강풍을 뿜는다.

**[진행 단계]**

- **Step 1. p/f 배지 획득 & P401 '소리 문 한 쌍'**
  - 바람문: 굳게 닫힌 돌문 옆에 낡은 종 — **f 배지**를 종에 부여해 타종(큰 소리) → 공명으로 개방.
  - 하피 둥지 길목: 잠든 바람 하피들 사이 통로 — **p 배지**를 자신의 발에 부여(발소리를 여리게) → 살금살금 통과. *같은 '소리 크기' 개념을 열기/숨기 양방향으로 학습.*
- **Step 2. P402 '크레센도 바람 계곡'** ([기믹문서] 원안 채택)
  - 힌트: 라라 "갈수록 커지는 건 크레센도. 그럼 줄이려면?"
  - 관찰: 3구간짜리 계곡. 구간마다 맞바람이 강해져(밀려남 가속) 맨몸으론 2구간부터 전진 불가. 바람 세기 게이지 표시.
  - 조작: **decresc.(디크레센도)** 획득 후 몸 앞에 방패처럼 장착 → 풍압 60% 감쇠 → 전진. 되돌아갈 때 **cresc.**를 장착하면 순풍 가속 활강(이동의 재미로 보상).
  - 피드백: 장착 중 바람 소리가 실제로 점점 작아진다/커진다.
- **Step 3. P403 '가라앉는 발판 호수'** — 밟으면 0.5초 만에 가라앉는 부낭 발판 지대. **스타카토** 부여 시 이동이 '짧게 통통 튀는 도약'으로 바뀌어 가라앉기 전에 건넌다(연속 탭 리듬 이동). 도감: "스타카토: 짧게 끊어서!"
- **Step 4. P404 '끊어진 파이프 다리'** — 절벽 양쪽의 이음 단자 사이를 **슬러(⌒)** 곡선으로 드래그해 잇는다 → 이음 다리 생성. 스팬 제한이 있어 '이을 수 있는 가장 가까운 단자 찾기'가 퍼즐. 도감: "슬러: 부드럽게 이어서."
- **Step 5. 정상 — 폭주 제어실** — f로 고정된 메인 밸브에 p 부여 + decresc.로 마무리 → 굉음이 점점 잦아들며(실시간 오디오 페이드) 폭풍 종식. *decrescendo를 귀로 확인하는 연출.*
- **Step 6. 클리어 컷신** — 셈여림 파트 복원: 같은 멜로디에 강약과 표정이 생긴다. 도감 특별 카드 "같은 노래도 셈여림이 있으면 표정이 달라져요"(복원 전/후 비교 재생 버튼). **오르간의 인장** 수여 + 산 지하 카타콤 입구 개방.
- **숨김 요소:** 황금 음표 #4 — cresc. 활강으로만 닿는 절벽 둥지.

**[보상 & flag]** p·f, cresc./decresc., 스타카토, 슬러, `seal_dynamics`, 셈여림 파트 복원, `catacomb_open`.

**[구현 힌트]** `equip(decresc) → wind.force *= 0.4` / `attach(staccato, player) → move = shortHop(1tile, air 0.2s)` / `slur(a, b) if dist ≤ maxSpan → spawnBridge`.

### 6.5 5막 — 불협화음의 카타콤 (종합 던전 + 중간 보스)

**[개요]** 던전 3구역 + 보스방 / 12~15분 / 입장: 인장 4개 (세뇨 관문 ④) / 획득: 세뇨의 인장, 화음 파트 / 학습: 장/단 3화음(3음이 표정을 결정), 강박·약박·당김음 — **지금까지 배운 전 기호의 종합 시험**

**[스토리]** ([기믹문서] 원안) 아름다운 화성이 흐르던 지하 신전이 기호가 뒤섞여 불협화음의 미로가 됐다. 깊은 곳에서 정박의 굉음이 울린다 — 다카포가 태엽 유적의 수호상을 뜯어가 만든 '엇박자 골렘'이다.

**[진행 단계]**

- **구역 1. P501 '화음 맞추기 문' 2연속** ([기믹문서] 원안의 이론 정밀화)
  - 문 A(밝음의 문): 바닥 근음 표시 'C'. 슬롯에 C·E♭·G가 꽂혀 있는데 문양은 장3화음을 요구한다.
    - 힌트: 라라 "가운데 음(3음)이 화음의 표정을 정해. 밝게 하려면 반음 위로."
    - 조작: E♭ 슬롯에 **♮** 부여 → C·E·G 완성.
    - 피드백: 완성 순간 C장3화음이 아르페지오→블록으로 재생되며 문이 밝게 개방. 틀린 조합은 탁한 불협화음.
  - 문 B(어둠의 문): 근음 'A', 슬롯 A·C♯·E, 요구는 단3화음 → C♯에 **♮** 부여 → A·C·E(단화음의 어둑한 울림과 함께 개방). *장/단의 소리 차이를 연속 비교 청취.*
- **구역 2. P502 '조표 함정 회랑 + 디스코드 잠입'** — 회랑 전체에 조표(♭1)가 걸려 모든 'B 수정'이 미끄럼 함정(B♭) 상태 + 디스코드 순찰. **♮으로 경로 확보 + 온쉼표 오라로 잠입**을 동시에 운용하는 복합 퍼즐(2막×3막 복습).
- **구역 3. P503 '굉음의 나선 계단'** — 아래로 갈수록 커지는 공명음이 플레이어를 밀어낸다 → **decresc. 방패 + 공명석에 p 부여**로 하강(4막 복습).
- **보스방. B501 — 중간 보스 '엇박자 골렘'** ([기믹문서] 원안 상세화)

| 항목 | 내용 |
|---|---|
| 무대 | 원형 아레나. 하단에 4박 타임라인 예고 UI 상시 표시 |
| 외형 | 메트로놈 + 바위 골렘. 등의 태엽이 약점 |
| 페이즈 1 (♩=90, 정박) | 1박·3박에 내리찍기(충격파 링). 타임라인 1·3칸에 주먹 아이콘 예고 |
| 공략 1 | ① 약박(2·4박)에 이동해 회피 ② **4분쉼표를 3박 칸에 삽입** → 그 사이클의 공격이 강제 휴지, 골렘이 헛스윙 후 1박 그로기 ③ 그로기 중 등 뒤로 돌아가 **스타카토 연타**로 태엽 타격 ×3 |
| 페이즈 2 (♩=120, 엇박) | 패턴이 당김음으로 바뀜: "쿵–(쉼)–쿵쿵–". 예고 아이콘을 읽고 대응해야 함. 신기술 '온박 프레스'(4박 전체 전방위 압박) |

| 실패 시 | 밀려나 아레나 입구에서 즉시 재도전(사망 없음) |
| 교육 효과 | 강박(1·3)과 약박(2·4), 당김음을 손가락으로 체득 ([기믹문서] 명시 목표) |

- **클리어 컷신** — 골렘 코어의 뒤틀린 태엽이 풀리며 정박으로 복귀, 골렘은 유적으로 돌아가는 수호상으로 정화된다. 템포: "…고향의 심장 소리야." 신전에 화성이 되살아나고 **화음 파트 복원**(오르골이 화음 반주를 얻어 곡이 최종형 직전이 된다). 세뇨가 직접 나타나 **세뇨의 인장** 수여: "이제, 표식 너머로." 하늘로 가는 빛의 다리 개방.
- **숨김 요소:** 황금 음표 #5 — 문 B를 '장화음으로도' 다시 맞춰보면 열리는 비밀 벽감(♮ 대신 ♯ 활용, 심화 실험 보상).

**[보상 & flag]** `seal_segno`, 화음 파트 복원, `bridge_of_light_open`, flag `dacapo_truth`(벽화로 다카포의 과거 공개).

**[구현 힌트]** `chordCheck(root, notes): intervals == [4,7] → major / [3,7] → minor` (반음 단위, 하드코딩 금지) / 골렘: `pattern = timeline queue; if timeline[i] == rest → skip attack[i]; golem.groggy(1beat)`.

### 6.6 최종막 — 무한 콘서트홀: 다카포전 (반복 기호 편)

**[개요]** 콘서트홀 3구역 + 최종 보스전 / 12~15분 / 입장: `bridge_of_light_open` / 획득: 페르마타, 코다, 피네 / 학습: 도돌이표·1/2번 괄호·페르마타·D.C.·코다·피네의 실제 진행 순서

**[스토리]** 다카포의 성은 같은 홀이 무한히 반복되는 공간. 여기서 반복 기호들은 먼저 '적의 권능'으로 체험되고, 플레이어는 그것을 읽고 역이용하는 법을 배운다.

**[진행 단계]**

- **Step 1. P601 '도돌이표 복도'**
  - 관찰: 복도 끝에 닿는 순간 시작점으로 되돌아간다. 벽을 살피면 𝄆 𝄇 표식 발견. 첫 회차 통로 위에 '1.' 괄호, 잠긴 옆문 위에 '2.' 괄호가 새겨져 있다.
  - 조작: 1번 괄호 구간 안 숨은 스위치(첫 회차에만 빛남)를 밟는다 → 루프로 되돌아간 **두 번째 회차**에 2번 괄호 문이 열려 있다.
  - 교육: "반복의 1회차는 1번 괄호로, 2회차는 2번 괄호로" — 진행 순서를 공간 동선으로 각인.
- **Step 2. P602 '샹들리에 홀 — 페르마타 획득'** — 거대 샹들리에가 4박마다 낙하해 길을 막는다. 홀 제단에서 **페르마타(𝄐)** 획득 → 낙하 예고 박 칸에 페르마타를 부여 → 그 박이 3배로 늘어나 샹들리에가 공중 정지 → 아래로 통과. 도감: "페르마타: 본래 길이보다 충분히 늘여요."
- **Step 3. B601 — 최종 보스 '혼돈의 지휘자, 마왕 다카포'** ([기믹문서] 3페이즈 원안 상세화)

무대: 대극장. 화면 하단에 **2마디(8박) 타임라인**으로 보스의 공격이 미리 예고된다([기믹문서] 채택).

| 페이즈 | 다카포의 패턴 | 파훼 (플레이어 행동) |
|---|---|---|
| **1. 페르마타 홀딩** | 지휘봉 음파 빔을 박자마다 스윙. 3사이클마다 8박째에 전멸기 '브레스' 예고가 타임라인에 점등(5박 전부터) | 브레스 박 도달 전에 **페르마타를 그 칸에 드래그** → 다카포가 그 박에 묶여 3초 경직 → 등 뒤로 돌아가 찢어진 악보 망토의 빛나는 마디를 타격 ×3 (페르마타 재사용 쿨 5박) |
| **2. 도돌이표 함정** | 무대를 독 안개로 분할하고 플레이어 구역에 𝄆 𝄇 → 구역 끝에 닿으면 시작점 워프 + 지속 피해 | 루프 구간 안 **1번 괄호 뒤 숨은 스위치**(첫 바퀴에만 노출)를 밟아 2바퀴째에 2번 괄호 통로 개방 → 통로 끝 제단에서 **코다(⊕) 획득** → 사용 시 루프를 건너뛰어 무대 2층 안전지대로 워프. 이후 다카포가 'D.C.'를 영창(8박 캐스팅, 맵 전체 리셋 시도) → **페르마타로 캐스팅 박을 늘려 끊고** 낙하 스타카토 반격 ×3 |
| **3. 피네** | HP 소진 후 폭주: 전 화면 무한 연타. 다카포 머리 위에 **빈 마지막 마디 슬롯**이 강조 표시 | 2층 제단(코다 도착점)의 봉인이 폭주 여파로 풀리며 **피네(Fine) 획득** — 이론대로 '코다가 피네로 인도'한다 → 빈 슬롯에 피네를 꽂는다 → **"D.C. al Fine"** 발동: 세계악보가 처음부터 연주되어 Fine에서 멈추고, 혼돈의 연주가 강제 종료된다 |

- 실패 시: 페이즈 시작 지점에서 재도전. 각 페이즈 파훼법은 템포가 1회 실패 후 힌트로 정리해 준다.
- 교육 효과: 도돌이표 → 1·2번 괄호 → 코다 → 피네, 그리고 D.C. al Fine("처음으로 돌아가 Fine에서 끝")의 진행을 전투 동선 그 자체로 체험.

### 6.7 엔딩 & 에필로그

1. **엔딩 컷신** ([개발문서] 25단계 구조): 피네가 꽂히는 순간 화면이 어두워지고 → 세계악보가 처음부터 끝까지, 지금껏 복원한 5개 파트 전부로 완주 연주된다 → 정화된 다카포가 '마에스트로 카포'로 돌아와 눈물의 지휘 → 마을 광장, 모든 NPC의 합주 → 타이틀 카드 "D.C. al Fine — 처음으로 돌아가, 끝을 향해."
2. **크레딧:** 획득한 악전 도감 카드가 크레딧과 함께 스크롤(전 기호 복습).

```text
Step 0  컨텍스트 로드 프롬프트 (새 AI 세션 시작용 — 본 설계서 요약 주입)
Step 1  프로젝트 셋업: 단일 index.html, Canvas, GameState 골격
Step 2  이동·충돌·카메라 (모바일 조이스틱 포함)
Step 3  타일맵 로더 & 맵 데이터 구조 (레이어 6종)
Step 4  상호작용·대화 시스템 (JSON 데이터 주도)
Step 5  기호 팔레트·부여/회수 시스템 + Web Audio 이론음
Step 6  프롤로그 콘텐츠 조립 (MQ01, 높은음자리표)
Step 7  1막 퍼즐 2종 (P101 청음, P102 오선 다리) + 화음 팡파르
Step 8  퀘스트 상태·flag 저장 (localStorage + 폴백)
Step 9  모바일 UI 폴리시·오디오 언락·임시 엔딩
Step 10 자가 테스트 총점검 & 배포 (v0.1 완성 기준표 대조)
```

---

*본 설계서는 [개발문서]의 "작지만 끝까지 완성" 원칙 아래, [기믹문서]의 모든 기믹·던전·보스 아이디어를 하나의 시나리오로 통합한 v1.0 청사진이다. Phase 1(v0.1)을 완성한 뒤 이 문서의 막 단위로 확장한다.*
